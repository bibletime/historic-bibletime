Kapitel 3. Installieren mittels BibleTime's Sourceocde

Dieses Kapitel beschreibt, wie der Quellcode compiliert wird. Diese Schritte treffen sowohl auf die Entwicklungsversionen als auch auf die veröffentlichten Versionen von BibleTime und Sword zu.

3.1. Installation von Sword und der Sword Module

Sword und die Sword Module sind die "Buchbibliothek" für BibleTime. Bitte lesen sie auch die Download-Seite für weitere Informationen über das Herunterladen von BibleTime.

Lesen Sie die "Systemvorraussetzungen" Seite um Sword 1.5 zu bekommen. Lesen Sie die "Verweise" Seite um Swords Module zu bekommen.

3.1.1. Installation von Sword 1.52

Ale Erstes müssen Sie die heruntergeladene .tar.gz Datei (wenn Sie die CVS Version benutzen, dann sind ihre Sourcen bereits entpacken. Sie können diesen Schritt also auslassen) in ein temporäres Verzeichnis entpacken. Machen Sie dies ungefähr folgendermaßen:

 mkdir /tmp
 cd /tmp
 tar -xzf sword-1.52.tar.gz
 cd /tmp/sword-1.52

(Die tar Datei wird in ein Verzeichnis ähnlich wie "sword-1.52" entpackt werden)

Bovor Sie versuchen Sword zu kompileren, müssen Sie Makefile.cfg abändern. (sie ist im obersten Verzeichnis des entpackten Quellcodes zu finden). Der änderbare Bereich ist gekennzeichnet, versuchen Sie nicht Bereiche der Datei zu verändern, die außerhalb des durch ##### Customizable stuff. Set it how you like it ##### Teiles liegen, außer wenn Sie wissen was sie machen.

Für viele Installationen werden die Voreinstellungen passen,. Wenn diese nicht passen, können die folgenden Schalter gesetzt werden.

Tabelle 3-1. Sword Makefile.cfg Schalter

SchalterBeschreibung
# compilerWenn Sie gcc installiert haben lassen Sie es unberührt, ansonsten geben Sie den von ihnen benutzen Compiler ein.
# shared librarySetzen Sie es auf 'yes'. Dies führt dazu, dass das "shared library" libsword.so erzeugt wird. Es muss ein Eintrag in /etc/ld.so.conf hinzugefügt werden, der auf libsword zeigt. /usr/local/sword/lib ist typisch.
# VCL libraryBenutzen Sie die Voreinstellung 'no', außer wenn Sie den Sword Installationsmanager benutzen (benutzen Sie dann 'yes').
# Debug Optionen usw. ...Die Voreinstellungen sollten funktionieren.
# endian, usw. Unterstützung für andere HardwareBenutzen Sie die Voreinstellung, wenn Sie auf einen Intel-Rechner kompileren. Wenn Sie andere Hardware benutzen, wählen Sie den richtigen Eintrag aus.
# zLib supportBenutzen Sie die Voreinstellung 'yes'
# pathsDie Voreinstellungen sollten ok sein, aber Sie können sie ändern, wenn Sie möchten. Wenn Sie den Installationsort der Bibliothek und der Include-Dateien ändern möchten, müssen Sie entscheiden, ob Sie sie in einem eigenen Verzeichnis wie /usr/local/sword/ oder ob Sie sie in einen bekannten Ort wie /usr/local/ installieren möchten. Wenn Sie die Bibliotheken in /usr/local/lib und die Include-Dateien in /usr/local/include installieren möchten, wählen Siee /usr/local.

Beginnen Sie nun durch das Eingeben von make und warten Sie bis es zu Ende ist. Danach müssen Sie die Dateien installieren. Das ist einfach. Geben Sie einfach make install ein.

3.1.2. Installation von Modulen

3.1.2.1. Wählen Sie ein Sword-Verzeichnis

Sword unterstützt zur Zeit drei Arten von Modulen: Bibeln, Lexika und Kommentare. Sie müssen die Dateien der Module in ihr eigenes Verzeichnis installieren. Sie müssen entscheiden, welches Verzeichnis ihr Sword Verzeichnis sein soll. Wenn Sie die Bibliotheken und Include-Dateien in ein eigenes Verzeichnis installiert haben benutzen Sie dieses als ihr Sword-Verzeichnis. Wenn Sie die Bibliotheken und Include-Dateien in einem bekannten Verzeichnis (z.B. /usr/local/sword/) installiert haben, so wählen Sie bitte einen anderen Ort für ihr Sword Verzeichnis.

3.1.2.2. Entpacken des heruntergeladenen Paketes

Als Erstes müssen Sie das heruntergeladenen Paket entpacken. Auf der Download-Seite von Sword können Sie zwischen einem Windows oder Raw Paket wählen. Wenn Sie einen "Nur Linux" Rechner benutzen so wählen Sie bitte das RAW Paket. Wenn Sie jedoch dummerweise ein selbstentpackende Programm für Windows heruntergeladen haben, so versuchen Sie bitte WINE zum Entpacken der Datei zu benutzen. Unter Umständen müssen die den Dateinamen von "_INSTALL.1" auf "_install.1" ändern, um das Setup zum Laufen zu bekommen.

3.1.2.3. Integration von Modulen

Legen Sie ein Verzeichnis mods.d innerhalb des Sword Verzeichnisses an. Kopieren Sie bitte due Datei [Modulname].conf in das mods.d Verzeichnis. Die .conf Datei ist normalerwiese in den Dateien des entpackten Modules enthalten. Verschieben Sie nun das Verzeichnis "modules" in ihr Sword Verzeichnis. Das "modules" Verzeichnis enthält ihre entpackten Datein des Modules. Im Beispiel ist /usr/local/sword das Sword Verzeichnis.

Die Verzeichnisstruktur sieht so aus:

 /usr/local/sword/ -|- mods.d/ -|- modul_1_name.conf
                    |           |- modul_2_name.conf
                    |
                    |- texts/rawtext/ -|-modul_1_name/dateien
                                       |-modul_2_name/dateien

3.1.2.4. Die Sword Konfigurationsdatei

Eine Sword basierte Anwendung muss wissen, wo die Module liegen. Sie benötigen eine kleine Konfigurationsdatei in /etc um das zu erreichen- Legen Sie die Datei /etc/sword.conf an. Die Datei muss nur die beiden folgenden Zeilen enthalten:

 [Install]
 DataPath=/usr/local/sword

Wenn Sie einen anderen Ort als /usr/local/sword im obigen Beispiel für oihr Sword Verzeichnis gewählt haben, so zeigen Sie mit DataPath auf ihr Sword Verzeichnis.

Starten Sie nun BibleTime und entspannen Sie sich ;-) Wenn Sie Probleme mit der Installation von Modulen haben, lesen Sie bitte die "FAQ" Seite. Wenn es dort keine Lösung für ihr Problem gibt, lesen Sie bitte auf der "Contacts" Seite wie man Hilfe bekommt.
