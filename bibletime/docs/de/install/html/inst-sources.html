Chapter 3. Installiation mit Hilfe des Quellcodes von BibleTime

Dieses Kapitel beschreibt, wie Quellen kompiliert werden. Diese Jobsteps beantragen die Entwicklung Version und die freigegebene Version von bibletime und von Sword.

Anlage

Sword- und Swordmodule sind die "bookbibliothek " für Bibletime. Sehen Sie bitte das Seite Download für weitere Informationen über das Downloading von BibleTime.

Haben Sie einen Blick an der Anforderungen Seite zum Erhalten der SWORD. Sehen Sie Seite Links, um SWORD's-Module zu erhalten.

Anlage

Diese Seite zeigt Klinge 1.5.1a. Für andere Versionen der Klinge, sind die Dateinamen unterschiedlich.

Zuerst müssen Sie die gedownloadete tar.gzdatei (wenn Sie die CVS-Version verwenden, werden Ihre Quellen bereits entpackt, also überspringen Sie diesen Jobstep), zu einem temporären Verzeichnis entpacken. Tun Sie etwas so: 

mkdir /tmp
cd /tmp
tar -xzf sword-1.5.1a.tar.gz
cd /tmp/sword-1.5.1a

(die Teerdatei hat in ein Verzeichnis entpackt, das etwas benannt wird, das sword-1.5.1a ähnlich ist),

Bevor Sie versuchen, Klinge zu kompilieren, können Sie ändern müssen Makefile.cfg (es ist im oberen Verzeichnis der entpackten Quellen). Das kundenspezifische Kapitel von Makefile.cfg wird gekennzeichnet, versucht nicht, Teile der Datei außerhalb des gekennzeichneten Bereiches zu ändern #####Customizable. Stellen Sie es ein, wie Sie es ##### mögen, es sei denn Sie wirklich wissen, was Sie tun.

Für viele Anlagen arbeiten die Rückstellungen. Wenn die Rückstellungen nicht arbeiten, können diese Schalter eingestellt werden.

Table 3.1. Sword Makefile.cfg switches

Switch Beschreibung
# compiler Wenn Sie GCC dieses allein verlassen lassen, betreten Sie andernfalls den Compiler, den Sie benutzen werden.
# shared library- Stellen Sie ' yes'. ein, das dieses die geteilte Bibliothek libsord.so veranläßt festgelegt zu werden. Ein Eintrag muß in die Datei /etc/ld.so.conf plaziert werden, das diese Punkte zu libsword. /usr/local/sword/lib typisch ist. Die geteilte Bibliothek ist notwendig für bibletime, um zu arbeiten, aber Klinge kompiliert ohne Fehler, wenn Sie nicht dies tun. Geben Sie hier acht.
# VCL library- Verwenden Sie die ' keine ' Rückstellung, es sei denn Sie die Klinge installieren Manager, dann benutzen ' yes benutzen.
# Debugging options etc... Die Rückstellungen sollten arbeiten.
# endian, etc. support for other hardware Verwenden Sie die Rückstellung beim Kompilieren auf einem INTEL. Wenn Sie andere Hardware haben, wählen Sie den korrekten Eintrag aus.
# zLib support Verwenden Sie die Rückstellung 'yes'
# paths Die Rückstellungen sollten okay sein, aber Sie können sie ändern, wenn Sie wünschen. Wenn Sie den installierstandort der Bibliothek und der umfassendateien anpassen möchten, müssen Sie entscheiden, wenn Sie sie in ein Extraverzeichnis wie /usr/local/sword/ installieren möchten, oder an einem allgemeinen Platz wie /usr/local/. oder Ihnen die Bibliotheken in /usr/local/lib installieren könnte und die umfassendateien in /usr/local/include/usr/local wählen.

Jetzt fangen Sie an, indem Sie make, zu kompilieren schreiben und warten Sie, bis er beendet ist. Nach diesem mssen Sie die Dateien installieren. Es ist einfach. Einfach make install .

Anlage

SWORD unterstützt aktuell 3 Arten Module: Bibeln, Verzeichnisse und Kommentare. Damit die Sword und bibletime zum Sein nützlich diese Module installiert werden müssen.

Entpacken Sie das gedownloadete Paket

Zuerst müssen Sie das gedownloadete Paket entpacken. Auf der Downloadseite der KLINGE können Sie wählen, wenn Sie Windows oder ROHES Paket erhalten möchten. Wenn Sie nur Linux Kasten bitte die ROHEN Pakete erhalten lassen. Aber wenn Sie versehentlich einen Windowselbst erhielten, der vollziehbaren Versuch mit WEIN, die Dateien zu extrahieren extrahiert. Möglicherweise müssen Sie den Dateinamen von _INSTALL.1 zu _install.1 ändern, um den Installation Betrieb zu erhalten.

Integrating modules

Zuerst müssen Sie das gedownloadete Paket entpacken. Auf der Downloadseite der KLINGE können Sie wählen, wenn Sie Windows oder ROHES Paket erhalten möchten. Wenn Sie nur Linux Kasten bitte die ROHEN Pakete erhalten lassen. Aber wenn Sie versehentlich einen Windowselbst erhielten, der vollziehbaren Versuch mit WEIN, die Dateien zu extrahieren extrahiert. Möglicherweise müssen Sie den Dateinamen von _INSTALL.1 zu _install.1 ändern, um den Installation Betrieb zu erhalten.

Verzeichnisstruktur sieht wie dieses aus: 

usr/local/sword/share/ -|- mods.d/ -|- module_1_name.conf
                   |           |- module_2_name.conf
                   |
                   |- texts/rawtext/ -|-module_1_name/files
                                      |-module_2_name/files
