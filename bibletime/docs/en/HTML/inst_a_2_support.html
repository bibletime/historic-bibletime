1.2. Supporting Programs

1.2.1. Determination of presence of supporting programs.

Start your package manager. Search for package qt. If the package description indicates that the version of qt on your system is 2.x.x (any version of qt 2) then qt is correct.

Now search for package KDE. If the package manager indicates that any version of kde2 is installed, then your kde is correct.

If qt or kde are not correct, you can download and install qt2 and kde2 (in that order) or you can use the version of bibletime with qt2 and kde2 compiled in.

Now check to see if sword is installed on your system. This is done by looking at /usr/local/sword If the directory is not present, sword will need to be installed. Sword is not provided with any distributions at this time, so unless you have previously installed it you probably will not find it.

You will also need to install Sword modules. The easiest way to see what is installed is to start bibletime and see what documents are available. All sword modules can be downloaded from crosswire.

1.2.2. Installation of supporting programs

1.2.2.1. qt2

If you install your qt2 from an rpm, nothing more needs to be done.

If you install qt2 from a troll tech source, you will need to unzip the file (using something like gnozip) and follow the directions in the file INSTALL.

1.2.2.2. kde2

If you install your kde2 from an rpm, nothing more needs to be done.

If you install kde2 from the source or CVS at kde.org, you will need to follow the installation directions for the file.

1.2.2.3. sword

If you install your sword from an rpm, nothing more needs to be done.

If you install sword from the source or CVS at kde.org, you will need to follow the installation directions in the INSTALL file.

1.2.2.4. sword modules

Sword modules can be downloaded from crosswire bible society.

A few sword modules are available as RPM's, however with the release of the installmgr program the sword modules in *.zip files can be installed as easily as the RPM's. Modules can be installed and added to an existing installation at any time.

See the section working with sword modules and installmgr.

1.2.3. Environment variables

1.2.3.1. Testing for environment variables and path

If you install from an RPM nothing needs to be done. If you compile yourself, you will need to set the environment variables.

in a bash shell, issue the command export, you will see a listing like this:

 declare -x KDEDIR="/usr"
 declare -x KDE_INITIAL_DESKTOP="1"
 declare -x LANG="en"
 declare -x LANGUAGE="en_US:en"
 declare -x LC_COLLATE="en_US"
 declare -x LC_CTYPE="en_US"
 declare -x LC_MESSAGES="en_US"
 declare -x LC_MONETARY="en_US"
 declare -x LC_NUMERIC="en_US"
 declare -x LC_TIME="en_US"
 declare -x LD_LIBRARY_PATH="/home/fks00/.kde/lib:/usr/lib"
 declare -x PATH="/bin:/usr/bin:/usr/bin/X11:
 /usr/local/bin:/usr/X11R6/bin:
 /usr/games:/home/fks00/bin:
 /usr/X11R6/bin:/usr/games:
 /usr/X11R6/bin:/usr/games"

Examine this file these items must be set:

 QTDIR=
 KDEDIR=
 KDEDIRS=

Also these items must be in your path command. If see error messages during the installation (or if bibletime is missing pieces, the most likely cause is improperly set environmental variables.

1.2.4. Setting environmental variables.

In a bash terminal in mandrake 7.2 use these commands (if you are using another distribution, or installed qt or KDE2 from a source, you will need to figure out the path names yourself):

 export QTDIR=/usr/lib/qt2
 export KDEDIR=/usr
 export KDEDIRS=/usr

If you need to add the directories to your path statement use something like

 export PATH=$KDEDIRS/bin:$PATH
 export PATH=$QTDIR:$PATH
 export PATH=$KDEDIR:$PATH

Now the environmental variables will be set for the terminal session.
