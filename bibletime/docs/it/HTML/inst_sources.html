Capitolo 3. Installazione dei sorgenti di Bibletime

Questo capitolo spiega la compilazione dei sorgenti. I punti qui descritti vanno bene per ambedue le versioni, la versione di sviluppo e la versione pubblica di BibleTime e Sword.

3.1. L'installazione di Sword e dei moduli Sword

Sword e moduli Sword sono la "libreria dei libri" per BibleTime. Vedi la pagina di download per ulteriori informazioni su come scaricare BibleTime.

Dai un occhiata alla pagina dei requisiti per ottenere Sword 1.5. Vedi i link per ottenere i moduli di Sword.

3.1.1. Installazione di Sword 1.52

Prima devi scompattare il file tar.gz scaricato (se usate la versione CVS i vostri sorgenti saranno già estratti perciò tralascia questo punto) in una directory temporanea. Fai qualcosa come: 

 mkdir /tmp
 cd /tmp
 tar -xzf sword-1.52.tar.gz
 cd /tmp/sword-1.52

(il file tar sarà estratto in una directora chiamata più o meno sword-1.52)

Prima di compilare Sword, forse devi modificare Makefile.cfg (è nella directory di livello più alto dei sorgenti estratti). La sezione da cambiare di Makefile.cfg è contrassegnata. Non modificare nulla fuori da quest'area ##### Customizable stuff. Set it how you like it ##### a meno che tu non sappia esattamente cosa stai facendo.

I default funzioneranno per la maggior parte delle installazioni. Altrimenti, ecco le parti che possono essere cambiate.

Tabella 3-1. Parti da cambiare in Sword Makefile.cfg

ParteDescrizione
# compilerSe hai gcc lasciate questo, altrimenti inserite il compilatore che stai usando.
# shared library-Default 'yes'. Questo fa sì che la libreria condivisa libsword.so sia generata. Una voce deve essere posta nel file /etc/ld.so.conf, indicante libsword. /usr/local/sword/lib è tipico.
# VCL library-Usa il default 'no' a meno che tu non stia usando il manager di installazione di Sword, nel qual caso usa 'yes'.
# Debugging options etc...I defaults dovrebbero essere giusti.
# endian, etc. support for other hardwareUsa il default se stai compilando su piattaforma Intel. Se hai una macchina diversa, seleziona la voce corretta.
# zLib supportUsa il default 'yes'
# pathsI default dovrebbero essere giusti, ma se vuoi puoi cambiarli. Se volete impostare la collocazione delle librerie e dei file include, devi decidere se vuoi installarli in una directory a parte come /usr/local/sword/ o in un posto condiviso come /usr/local/. Puoi anche installare le librerie in /usr/local/lib e i file include in /usr/local/include. Allora scegli /usr/local.

Adesso inizia la compilazione con make e aspetta finché non è terminato. Dopo questo devi installare i file. È' facile. Semplicemente scrivi make install .

3.1.2. L'installazione dei moduli

3.1.2.1. Scegli una directory Sword

Attualmente Sword include 3 tipi di moduli: Bibbie, lessici e commentari. Devi decidere quale directory sarà quella di Sword, per poterci installare i moduli. Se hai installato le librerie e i file include in una directory a parte, usa quella (p.e. /usr/local/sword/). Se invece li hai installati in una directory condivisa, come /usr/local/, allora scegliene un'altra per i moduli di Sword.

3.1.2.2. Aprite il pacchetto scaricato

Per prima cosa devi scompattare il pacchetto scaricato. Sulla pagina di scaricamento di Sword puoi scegliere tra pacchetti Windows o RAW. Se hai una macchina "solo Linux" allora scegli i pacchetti RAW. Se però ti capita di scaricare un pacchetto Windows prova ad estrarre i file con WINE. Può darsi che occorra cambiare il nome del file "_INSTALL.1" in "_install.1" per far funzionare il setup.

3.1.2.3. Integrare i moduli

Crea la directory mods.d dentro la directory Sword. Copia il file [nome_del_modulo].conf nella directory mods.d. Il .conf file è normalmente tra i file estratti. Poi spostate tutta la directory "modules",la quale contiene i vostri file di dati del modulo, nella directory Sword. Nell'esempio /usr/local/sword è la directory Sword.

La struttura della directory è questa:

  /usr/local/sword/ -|- mods.d/ -|- module_1_name.conf |
 |- module_2_name.conf
                    |
                    |- texts/rawtext/ -|-module_1_name/files
                                       |-module_2_name/files

3.1.2.4. Il file di configurazione di Sword

Un'applicazione basata su Sword deve sapere dove si trovano i moduli. Per questo hai bisogno di un piccolo file di configurazione in /etc. Crea il file /etc/sword.conf. Nel file ci devono essere soltanto le due linee seguenti: 

 [Install]
 DataPath=/usr/local/sword

Se scegli un luogo diverso da /usr/local/sword nell'esempio sopra per la tua directory Sword indica come DataPath la directory scelta.

Adesso avviate BibleTime e apprezzate ;-) Se hai problemi con l'installazione dei moduli leggi per favore la pagina "FAQ". Se non c'è una soluzione per il tuo problema, consulta per favore la pagina "Contact" per ottenere aiuto.
